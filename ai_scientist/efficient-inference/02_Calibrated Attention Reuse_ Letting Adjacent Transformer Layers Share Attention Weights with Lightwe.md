## 名称
calibrated_attention_reuse

## 标题
校准注意力重用：通过轻量级校正让相邻Transformer层共享注意力权重

## 简短假设
从已计算层中重用完整的注意力权重矩阵（softmax之后），并结合轻量级、逐层校准（例如，在注意力值上应用学习到的逐头仿射变换），可以在显著减少因不必要的 QK^T 和 softmax 重计算导致的 FLOPs 的同时，取得几乎相同的任务性能。与朴素注意力重用（Shim et al. 2023）和选择性 token 索引重用（HyLRA）相比，本方法在准确率-效率权衡上表现更优。

## 相关工作
Shim et al. (2023) 探索了注意力图重用，通过将层分组来共享一个未经修改的注意力图，但未引入任何逐层自适应调整，导致在激进重用时准确率下降。HyLRA (2026) 仅跨层重用前 k 个 token 索引，这在内存上更友好，但选中的 token 仍需重计算注意力，且丢失了细粒度的权重信息。我们的方法——校准注意力重用（CAR）——保留了完整的 softmax 注意力图，并应用一个微小的逐层、逐头校准网络（例如，对加权的值求和输出应用学习到的尺度和偏置）来补偿层间分布偏移。这种校准引入的计算量极小（不到单层 FLOPs 的 0.5%），同时允许高重用率。其他工作如 RePaViT 专注于 FFN 层的结构重参数化，而非注意力重用。我们的方案是首次将完整注意力权重重用与高效校准相结合，以保持准确率。

## 摘要
自注意力机制是 Transformer 模型中的主要计算瓶颈，尤其是对于长序列，因为计算 QK^T 和 softmax 的代价是二次的。虽然已有工作表明相邻层的注意力图通常高度相似，但朴素重用（直接将注意力图从一层复制到下一层）会导致明显的准确率下降，因为特征表示在层间会演化。我们提出校准注意力重用（CAR），一种系统地跨连续层块共享注意力权重矩阵的方法，同时插入轻量级的、学习到的逐头校准模块。具体而言，在一个 N 层块中，我们仅在第一个层计算完整注意力，并在随后的 N-1 层中重用相同的 softmax 注意力图。对于每个重用层，在将注意力图应用到该层自己的值之前，我们通过一个逐头仿射变换（一个可学习的尺度和偏置，或一个隐藏大小为8的微小2层MLP）来调整注意力值的输出，使其匹配完整注意力计算的分布。校准参数在所有输入 token 间共享，且评估代价极低。我们在 BERT-base（GLUE）和 GPT-2（Wikitext-103）上验证了 CAR，涵盖预训练和微调场景。实验表明，当重用块大小为2（即使用第 L 层的注意力用于第 L+1 层）时，CAR 保留了超过 99% 的原始性能，同时在重用层上节省了约 50% 的注意力 FLOPs；在块大小为4时，节省达到 75%，且大多数任务上准确率损失不到 1%。CAR 在更高重用率下一致优于朴素重用和 token 索引重用基线。我们的工作表明，轻量级、可学习的校正是释放有效注意力重用的关键，为高效的 Transformer 推理提供了一条简单且硬件友好的路径。

## 实验
**模型与任务：** BERT-base（GLUE：SST-2、MRPC、QQP、MNLI）和 GPT-2（Wikitext-103 语言建模、SQuAD v2 问答）。

**实现细节：** 我们将 Transformer 层分为大小为 K 的组（例如，K=2,3,4）。在每个组中，仅第一层计算完整注意力（QK^T 和 softmax）。对于剩余的 K-1 层，我们重用来自第一层的注意力矩阵 A。重用层首先计算自己的 Q、K、V 投影。然后，不计算自己的 QK^T，而是直接使用存储的 A（即 Q_first * K_first^T 的 softmax）。但关键在于，在将 A 与其 V 相乘之前，我们应用一个校准模块：对于每个头，对 A*V 的输出施加一个学习到的尺度向量 s（大小为 d_head）和偏置向量 b（大小为 d_head）：输出 = s * (A * V) + b。参数 s 和 b 特定于每个重用层和头，并在所有 token 间共享。我们还尝试了使用一个微小的 2 层 MLP（隐藏大小为 8）替代仿射变换，但仿射变换已经足够有效。校准参数初始化为恒等（scale=1, bias=0），并与原始任务损失进行端到端训练。

**训练：** 对于 BERT，我们在每个 GLUE 任务上微调 3 个 epoch，学习率为 2e-5。对于 GPT-2，我们在 Wikitext-103 上从头预训练 100k 步，并在 SQuAD 上微调。我们对比了：(1) 完整基线（无重用），(2) 朴素重用（无校准），(3) token 索引重用（HyLRA 风格，每层重用前 k 个 token 索引，k 设置为匹配 FLOPs），(4) 每隔一层无注意力（在重用层完全跳过注意力）。

**评估指标：** 任务准确率（GLUE、SQuAD F1）、困惑度（Wikitext-103）、FLOPs 降低（理论值与自定义内核测量值）、A100 上批量大小为 1 的端到端推理延迟，以及添加的校准参数数量。

**消融实验：** (a) 改变块大小 K = 2,3,4,6。(b) 比较仿射校准、MLP 校准和无校准。(c) 分析跨层注意力图的相似性（注意力向量的余弦相似度），以验证重用的可行性。(d) 分别对深层和浅层应用 CAR。

## 风险因素与局限性
- **校准开销：** 仿射校准增加了少量参数和 FLOPs。我们必须确保它不会抵消节省。对于12层模型，块大小为2，我们增加约36k个参数（12个重用层 * 12个头 * 2*64个scale/bias值），这可以忽略不计。每层 FLOPs：每个 token 2*d_head，远小于注意力。
- **注意力分布漂移：** 在非常深的模型中或者具有不同角色（早期 vs 后期）的层中，注意力图可能显著发散。我们通过仅在连续块内重用并允许块大小为1（无重用）来处理早期/后期层来缓解。我们还建议通过验证自适应选择块大小。
- **训练稳定性：** 校准参数可能干扰训练。我们使用谨慎的初始化（恒等）且无额外调度；在初步实验中未观察到问题。
- **硬件效率：** 跨层重用相同注意力矩阵需要将其存储在内存中。这使每层的内存占用增加 O(L^2)。对于长序列（例如 4K token），12个头，注意力矩阵大小为 4K*4K*12*4 字节 ~ 768 MB 每个重用层，这可能过高。我们通过在局部组内重用并在组计算后释放内存，或者使用重计算来缓解。这是一个关键限制；我们讨论了可能的系统级解决方案（例如，廉价地重计算注意力图而非存储）。我们将主要关注中等序列长度（<=1024），其中存储是可行的；对于更长序列，我们建议使用轻量级重计算（例如，使用缓存的 query）。
- **与 FlashAttention 的比较：** 我们的方法与内存高效的注意力实现是正交的；节省来自避免 QK^T 和 softmax，而非更好的内存访问。
