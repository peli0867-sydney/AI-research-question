## 名称
latent_query_augmentation

## 标题
潜在查询增强：在检索增强生成中从大语言模型内部状态检索信息

## 简短假设
生成过程中语言模型的内部隐藏状态不仅编码了模型已知的信息，也编码了它需要知道的信息。通过将这些状态投影到检索查询空间，我们可以生成最适合填补模型特定知识空缺的查询，从而相比仅使用输入查询或依赖置信度阈值的现有方法，实现更精确、更符合事实的生成。这种方法的独特之处在于它利用了表示的内容（而不仅仅是其不确定性），并且不需要手工设计的启发式规则或额外提示。

## 相关工作
现有的自适应检索工作（如 Self-RAG、SeaKR、FLARE）使用显式的反思令牌、置信度分数或预定义触发器来决定何时检索。它们不使用生成器的实际隐藏状态来构建查询。像 REPLUG 这样的方法使用 LLM 的似然度来加权检索到的文档，但仍然使用原始查询进行检索。查询生成方法（如 HyDE）会生成假设性文档，但不会以生成器的状态为条件。我们的提议直接将隐藏状态映射到查询嵌入，从而在生成器的知识空缺与检索过程之间建立紧密、可学习的耦合。这是表示学习在 RAG 中的一个新颖应用。

## 摘要
检索增强生成（RAG）通常基于用户输入查询检索段落，然后统一用于生成。然而，随着生成的进行，模型的信息需求会发生变化：早期令牌可能需要宽泛的上下文，而后续令牌可能需要具体事实。现有的自适应方法决定了何时检索，但仍使用原始查询，这限制了检索的相关性。我们提出潜在查询增强（LQA），一种训练轻量级投影网络的方法，该网络可将 LLM 在每个生成步骤的隐藏状态映射为检索查询嵌入。然后使用该嵌入检索额外段落，直接满足模型当前的知识需求。投影器通过 REINFORCE 目标进行训练，该目标奖励下游答案的准确性，同时惩罚冗余或不相关的检索。我们在多跳问答（MuSiQue、HotpotQA、2WikiMultiHopQA）和长篇问答（ASQA）上评估了 LQA。与标准 RAG 相比，LQA 在 F1 分数上提升了 3-5%，在多跳问题上优于自适应基线（Self-RAG、SeaKR），同时将检索调用次数减少了 30%。分析表明，LQA 生成的查询在语义上多样化，并且针对的是缺失的关系信息，而不仅仅是实体。

## 实验
- 1. 在冻结的 LLM（Mistral-7B）的最后一个隐藏状态之上，为每个解码步骤训练一个投影器（2 层 MLP，256 维）。投影器输出一个嵌入，用于查询稠密检索器（Contriever-MS MARCO）。
- 2. 使用 REINFORCE，奖励 = 最终答案的 F1 分数减去每次检索的成本（例如 0.1 * 检索次数）。在 HotpotQA 的子集（500 个样本）上训练。
- 3. 基线方法：（a）标准 RAG（使用输入查询检索一次），（b）Self-RAG，（c）SeaKR，（d）FLARE，（e）REPLUG。使用相同的检索器和 LLM 主干网络。
- 4. 数据集：MuSiQue、HotpotQA、2WikiMultiHopQA（多跳，指标：F1、EM）；ASQA（长篇，指标：MAUVE、通过 NLI 评估的事实性）。
- 5. 评估指标：准确率、每次查询的检索调用次数、检索查询的多样性（通过嵌入距离衡量）、以及相关性的人工评估。
- 6. 消融实验：（a）从不同层投影（早期层 vs 晚期层），（b）为每个注意力头使用独立的投影器，（c）用监督学习替代 REINFORCE，使用 oracle 查询（来自黄金证据）。
- 7. 定性分析：展示一些例子，其中 LQA 生成诸如“爱因斯坦出生国家的首都是什么？”这样的查询，此时模型需要解决多跳依赖。

## 风险因素与局限性
- 1. 训练不稳定性：基于每步动作的 REINFORCE 可能具有高方差。缓解措施：使用移动平均基线并裁剪奖励。
- 2. 额外开销：每次检索调用时，投影器推理会增加一次小前向传播，但我们限制每次生成最多检索 3 次，从而控制成本。
- 3. 隐藏状态可能不包含足够强的信号来处理复杂的多跳推理。缓解措施：使用中间层（例如 32 层中的第 16 层），这些层可能编码了更多语义内容。
- 4. 对域外查询的泛化能力：投影器可能过拟合训练模式。需要在未见过的数据集上进行测试（例如 TriviaQA、PopQA）。
- 5. 对检索器质量的依赖：如果检索器索引不包含所需知识，投影器也无法发挥作用。但这是 RAG 的普遍局限性。
